O que é Git e GitHub?

Git é um sistema de controle de versões distribuído e amplamente utilizado para rastrear mudanças em arquivos durante o desenvolvimento de software. Ele permite que vários desenvolvedores trabalhem simultaneamente em projetos, gerenciando suas modificações de maneira eficiente. O Git mantém um histórico detalhado das alterações, facilitando o trabalho colaborativo e a revisão de código.

GitHub é uma plataforma de hospedagem de código-fonte que utiliza o Git como seu sistema de controle de versão. Além de armazenar projetos, o GitHub oferece ferramentas poderosas para colaboração, como controle de versão, gerenciamento de problemas, solicitações de pull (pull requests) e integração contínua. Ele se tornou essencial para desenvolvedores que buscam compartilhar e colaborar em projetos de software de forma eficaz.

Como Usar Git e GitHub

1. Configuração do Git

Primeiro, você precisa instalar o Git no seu computador. Você pode baixá-lo e seguir as instruções de instalação do site oficial do Git: https://git-scm.com. Depois de instalar o Git, é importante configurar o seu nome de usuário e endereço de e-mail, que serão associados aos seus commits.

2. Configuração do GitHub

Para começar a usar o GitHub, siga os passos abaixo:

  1. Crie uma conta no GitHub em https://github.com/ se ainda não tiver uma.
  2. Para criar um repositório, logue na sua conta do GitHub e clique em "New" no canto superior direito da página.
  3. Preencha o nome do repositório, descrição (opcional), e escolha se quer um repositório público ou privado.
  4. Clique em "Create repository".
  5. Você está pronto para compartilhar seus códigos com essa incrível comunidade.

Comandos Git - Uso na Prática

Comando Descrição Exemplo
git init Inicializa um novo repositório Git. git init
git clone Clona um repositório existente. git clone https://github.com/usuario/repo.git
git branch Cria, lista ou exclui branches. git branch nova-branch
git checkout Altera para um branch específico. git checkout nova-branch
git status Exibe o status das alterações. git status
git diff Mostra as diferenças entre commits. git diff
git add Adiciona arquivos à área de staging. git add arquivo.txt
git commit Salva as alterações no repositório. git commit -m "Mensagem do commit"
git push Envia commits para o repositório remoto. git push origin main
git pull Atualiza o repositório local com mudanças do repositório remoto. git pull origin main
git revert Reverte um commit específico. git revert <hash-do-commit>
git merge Mescla branches. git merge nova-branch
git stash Armazena alterações temporariamente. git stash
git rm Remove arquivos do repositório. git rm arquivo.txt
git config Configura opções do Git. git config --global user.name "Seu Nome"
git config --global user.email "seuemail@example.com"
git reset Desfaz commits locais. git reset --hard <hash-do-commit>
git remote Gerencia repositórios remotos. git remote add origin https://github.com/usuario/repo.git
git fetch Busca mudanças de um repositório remoto. git fetch origin
git show Exibe informações sobre qualquer objeto Git. git show <hash-do-commit>
git help Exibe o manual do Git. git help
git rebase Aplica commits de um branch a outro. git rebase nova-branch
git blame Mostra quem modificou cada linha de um arquivo. git blame arquivo.txt
git tag Marca pontos específicos na história do repositório. git tag -a v1.0 -m "Versão 1.0"
